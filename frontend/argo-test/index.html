3. Notes techniques

  • Backend par défaut : …
  • Prompt système : chargé à chaque connexion via GET /api/prompt?product=<id>
  • BM25 : exécuté côté serveur, exposé via POST /api/briefing (tool calling)
  • Coach : appel async toutes les 2 user turns via POST /api/strategist
  • UI Director : appel à chaque user turn via POST /api/ui-director
  • Les conversations sont sauvées en SQLite (/api/save-conversation) + Google Sheets (via Apps Script)
